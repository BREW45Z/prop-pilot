# USER FLOW

## First-Time User Flow

1. User opens Prop Pilot.
2. User sees a minimal landing screen.
3. User reads the short product promise.
4. User clicks Start Calculating.
5. User enters the dashboard with a sidebar.
6. User understands the MVP tools are Risk Calculator and Daily Drawdown.
7. User selects one sidebar tool.
8. User follows the guided wizard.
9. User enters values and reads a compact result report.
10. User can copy the summary or open a Share Card and download a PNG.

## Returning User Flow

1. User opens Prop Pilot before or during trading.
2. User clicks Start Calculating.
3. Saved calculator values load locally when available.
4. User enters updated account or daily P/L values.
5. User checks risk, drawdown, status, and Trading Insight.
6. User decides whether to trade, reduce risk, or stop.

## Calculator Usage Flow

Risk Calculator:

1. Select Risk Calculator from the sidebar.
2. Enter Account Size.
3. Enter Risk %.
4. Enter Stop Loss.
5. Enter Dollar Value per Pip.
6. Review Risk Amount.
7. Review Position Size.
8. Copy Summary or open Share Card / PNG Download.

Daily Drawdown Calculator:

1. Select Daily Drawdown from the sidebar.
2. Enter Start of Day Balance.
3. Enter Daily Loss Limit %.
4. Enter Current Day P/L.
5. Enter Open Trade Risk.
6. Review Max Daily Loss.
7. Review Loss Used.
8. Review Remaining Room.
9. Review Status and Trading Insight.
10. Copy Summary or open Share Card / PNG Download.

## Active MVP Layout Flow

1. Landing screen appears first.
2. User clicks Start Calculating.
3. Sidebar stays visible on desktop.
4. User chooses Risk Calculator or Daily Drawdown.
5. Main content stays centered and compact.
6. Reports use wide horizontal grids instead of tall stacked cards.
7. Normal laptop screens should avoid unnecessary vertical scrolling.
8. Back to Home returns the user to the landing screen.

## Trading Insight Flow

Daily Drawdown results include a Trading Insight section.

- Messages are short, direct, and discipline-focused.
- Messages change based on drawdown status:
  - Healthy
  - Close to Breach
  - Breached
- A message randomizes when the user reaches the result screen.
- The message remains stable while the user is reading.
- The product does not provide market predictions, trade signals, or buy/sell instructions.

## Future Dashboard Flow

1. User selects a prop firm challenge.
2. User sees challenge rules and account progress.
3. User uses calculators based on the challenge rules.
4. User logs trades or daily results.
5. User reviews risk status before continuing to trade.
6. User eventually sees a Prop Health Score as the signature account safety signal.
